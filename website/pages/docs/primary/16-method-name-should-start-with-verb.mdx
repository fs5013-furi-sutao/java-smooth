<p class="goal">
  クラス名は名詞始まり、メソッド名は動詞始まりという命名が一般的なため、
  開発現場ではそこが守られていないと、違和感のあるコードになってしまうことを知る
</p>

## 良いメソッド名

良いメソッド名は，名前から機能が想像できます。
メソッドは処理を行うので、動詞または動詞句を割り当てます。

動詞句とは「基本動詞＋前置詞または副詞」のことです。「メソッド名は動詞（＋名詞）で構成される」と覚えておけば OK です。

| メソッドの役割             | メソッド名                    |
| :------------------------- | :---------------------------- |
| 値の取得に関するメソッド   | getXXX                        |
| 値のセットに関するメソッド | setXXX                        |
| 生成に関するメソッド       | create，build，make，genarate |
| 初期化に関するメソッド     | initialize，setupXXX          |
| 破棄に関するメソッド       | destroy，dispose              |
| 状態の確認に関するメソッド | contains，exists              |

永続性（DB などでデータを保存しておくこと）の 4 つの基本機能である `CRUD` は、Create（作る）、Read（読む）、Update（更新する）、Delete（削除する）の頭文字です。これらの動詞を使えば、ある程度のメソッド名は網羅できます。

「顧客データを生成する」であれば「CreateCustomerData」になりますし、「顧客データを削除する」であれば「DeleteCustomerData」のように命名します。

## たくさん試す、たくさん読む

実際は微妙なニュアンスを表現したいので、メソッド名には数多くの動詞が使われます。

たくさんコードを書いて、どんなメソッド名、動詞がしっくりくるのか？というベストプラクティスを見つけていくのが、イコール、コーディングの上達とも言えます。

また、たくさんのサンプルコードを読んで、優れたプログラマはどんな感覚でメソッド名を付けているのかというのを、数多く経験していくことも重要です。

## クラス名とメソッド名で、英語の文章として意味が通るように

インスタンスメソッドは，「⁠dateFormat.parse」のようにオブジェクト名と組み合わせて呼び出されます。
そのため、組み合わされたときに重複なく意味が通る名前が、良いメソッド名の命名方法です。クラスメソッドとクラス名も同様です。

例えば、Dog クラスに bark() メソッド（ほえる）があれば、インスタンスメソッドの呼び出しは、
`dog.bark()` になりコードが文章として読めるので、始めて処理を見ても、処理が読みやすくなります。
