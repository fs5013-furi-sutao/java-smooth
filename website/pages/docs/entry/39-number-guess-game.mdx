# 39. 作成課題 ⭐ 数当てゲーム

<p class="game-content">
  コンピュータがランダムで決めた数字（0 ～ 99）を、プレイヤーが当てるゲーム
</p>

<strong style="color: #ff0000;">
  この課題は作成が完了したら、教育担当者にコードレビューを依頼してください
</strong>

## 内容

コンピュータがランダムで決めた数字（0 ～ 99）を、プレイヤーが当てるゲームです。

数字を答えられるは５回までです。

プレイヤーが数字を入力すると、その数字がコンピュータが決めた数字が違っている場合は、大きいか小さいかを教えてくれます。

見事ピタリと当てることができればプレイヤーの勝ちです。

**実行例**: <span style="color: #ff0000;">**不正解**</span>

```
数字を当ててみてね。

答えられるのは５回までだよ。

1回目：43
もっと大きい数字だよ

2回目：80
もっと小さい数字だよ

3回目：60
すごい！！3回で当てられちゃった！
```

**実行例**: <span style="color: #00d176;">**正解**</span>

```
数字を当ててみてね。

答えられるのは５回までだよ。

1回目：43
もっと大きい数字だよ

2回目：76
もっと大きい数字だよ

3回目：90
もっと小さい数字だよ

4回目：81
もっと大きい数字だよ

5回目：87
もっと小さい数字だよ

残念！！ 正解は 85 でした！
```

## チャレンジ

以下の改造を行ってみよう。

1. 10 回まで数字を入力できるようにする
2. 3 桁の数字（0 ～ 999）を当てる仕様に変える
3. 数字が遠く離れているか近いかでメッセージを変える

### コンピュータの数字をどうやって決めればよいのか？

ランダムな数字のことを、プログラミング用語では「乱数」と呼びます。

Java には乱数(ランダムな数字)を取得するための方法があります。

キーワード「 [java ランダム 範囲](https://www.google.com/search?q=java+ランダム+範囲) 」で Google 検索してみましょう。

### ユーザーに数字を入力させるには？

キーワード「[Java キーボード入力 数字](https://www.google.com/search?q=java+キーボード入力+数字)」で Google 検索してみましょう。

### 全体の処理の流れ

#### 1. 初期化

1. ランダムクラスの準備をする

2. キーボード入力の準備をする

#### 2. コンピュータの数字を決める

変数 number を用意し、値を 0 ～ 99 の中からランダムで決める

#### 3. 「数字を当ててみてね。答えられるのは５回までだよ。」を表示する

#### 4. 5 回解答させるので for を使って繰り返す

1. 「〇〇回目」を表示する

2. 変数 inputNumber を用意し、入力した数字を inputNumber に代入する

3. ユーザーが入力した数字(inputNumber)を表示する

4. number と inputNumber が等しいとき、正解表示を行う

   1. 「すごい！！〇〇回で当てられちゃった！」を表示する

   2. return でプログラムを終了する

5. number を inputNumber が異なるとき

   1. number より inputNumber が大きいとき、「もっと小さい数字だよ」を表示する

   2. number より inputNumber が小さいとき、「もっと大きい数字だよ」を表示する

#### 5. 5 回解答しても正解できなかったとき、「残念！！正解は〇〇でした！」を表示する

#### 6. 終了処理を行う(キーボード入力処理の終了)
