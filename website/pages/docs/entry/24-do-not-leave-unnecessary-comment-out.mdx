# 24. 不要なコメントアウトを残さない

<p class="goal">不必要なコメントアウトを残さないようにコーディングできること</p>

## 学習

### 不要なコメントアウトを残すのは悪

レビュアー（レビューをする人）として、コードレビューをすると、コメントアウトした処理が残ったままであるのを目にすることが多々あります。

コメントアウトした処理を残すことは、不要な記述が残ることで、誤解を招いたり、コードの可読性を落とすので、良くありません。

### 今はバージョン管理がある

開発現場によっては、変更前のソースコードをコメントアウトにして残す文化があります。これは、Subversion や Git などのバージョン管理システムが利用される前の文化の名残です。Git などのバージョン管理を利用している場合は、こうした文化は無用です。

コメントアウトして不要となった処理は、なるべく早い段階で削除する習慣を付けましょう。
