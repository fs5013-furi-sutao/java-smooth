# 51. 悪い習慣 😈 見ればわかることをコメントに書く

<p class="goal">
  見ればわかるようなことを、わざわざコメントに書くのはムダであり、
  コードを読みにくくしてしまうことを理解する
</p>

** この項目には、ミニ課題はありません **

## 学習

### 見ればわかることをわざわざコメントに書くのは悪

コメントがなくても読み取れることを、わざわざコメントとして書くのは良くありません。
例えば、以下の処理は、やりたいことの意図が読み取れると思います。

```java
int time = endTime - startTime;
```

そこにわざわざコメントを書くというのは、コードに情報量をいたずらに増やすだけで、大事な部分を読みにくくしてしまう行為になります。

```java
// 終了時間から開始時間から所要時間を計算する
int time = endTime - startTime;
```

ひどい例ですが、実際の業務で使われているコードでも、
while 文の前に`// ループ処理` というようなコメントが散見されるものもありました。

```java
// ループ処理
while (!isAllTaskfinished) {
    /* 各タスク状態のチェック */
}
```

誰が見てもループ処理してることは明白です。
無意味なコメントを書くとゴミとなるだけなので、見ればわかることをコメントに書くべきではありません。

### 実は変数名の付け方が悪いのかも？

以前の項目でも「不吉なにおい」として紹介したことに「変数名にコメントを付ける」というのがありました。

いや、そのコメントは必要なんですよ、というものの中には、変数名が悪いからコメントが必要になっているケースも少なくありません。

例えば、以下のコメントは一見、必要なコメントのように見えます。

```java
// 30人の生徒の得点を表す配列を用意する
int[] points = new int[30];
```

しかし、これは `points` という変数名が何を表すか、`30` という要素数が何を表すかが分かりにくいため、コメントが必要になっています。
これは、以下のように改善（リファクタリング）することで、コメントが不要になります。

```java
final int NUMBER_OF_STUDENT = 30;
int[] studentScores = new int[NUMBER_OF_STUDENT];
```

心に刻むべきは、「コメントで悪いコードは良くならない」という事実です。

### 多すぎるコメント

時々、初学者の中で、丁寧にコメントを付けるあまり、コードよりコメントの量が多くなっているケースも見かけます。

これは、教科書などに掛かれている「処理が分かりやすくなるようにコメントを利用してください」
というアドバイスに従ってのことだと思われます。

プログラムには必ずコメントを書けと指導されるケースもあると聞きます。

しかし、コメントを詳細に書けば、処理が分かりやすくなるというわけではありません。
次の例を見てください。

```java
// ジュースストックから指定したジュースのラックを取得する。
// 取得したラックはrackに格納し、後続処理で使用する。
Juice rack = juiceStock.getRack(juice);
// ラックに在庫がある場合のみ処理する
if (rack.isInStock()) {
    // ラックからジュースを取り出す
    // ジュースが取り出せない場合は例外となるが、
    // 先に在庫を確認しているため通常は起こらない
    rack.remove();
    // ジュースが取り出せたら購入できるので、
    // ジュースの単価を購入処理に渡して処理する
    moneyFlow.purchase(juice.getPrice());
}
```

コードを読めばわかることも書いていますが、間違っていないため読む価値があるように思われてしまいます。
しかし、コメントがあることで逆にコードを読む速度が低下してしまいます。

コメントは必要な場所に必要なだけが鉄則です。それが「コメントは適宜つけましょう」の意味です。

### その後のメンテナンス工数も考える

コメントは、処理内容に改修が入った際にも、その改修に合わせてコメントも修正しなければいけません。

つまり、コメントが増えるということは、その後のメンテナンスの手間も増えてしまうということです。

その点でも、無駄なコメントは、初期の開発時になくすべきです。

## まとめ

見てすぐに処理の意図が読み取れるような箇所にわざわざコメントを書くのは無駄な行為です。
無駄にコードを膨張させ、重要な処理を読みにくくしてしまう可能性もあります。

一見、このコメントは必要だよな、と思っても、実は変数名が悪いためにコメントが必要になっているケースもあります。
そのため、無駄にコメントが多いなと感じたら、変数名やメソッド名が適切かどうかを再確認する必要もあります。

また、業務で使うようなコードにおいては、長く運用してくことを考えると、コメントには改修時に発生するメンテナンス工数が掛かってきます。

以上の理由から、コメントは、そもそものコードを見直した上で、必要最低限のみコードに付与するべきです。
